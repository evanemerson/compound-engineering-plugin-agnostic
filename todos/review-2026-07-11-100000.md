## Findings

### 1
- status: applied
- severity: P2
- agent: adversarial-reviewer
- category: Git safety / user-work loss
- confidence: 80
- action_class: judgment
- file: plugins/cepa/commands/compound-refresh.md
- lines: 267-272
- title: Dirty refresh-touched file blends user edits into the bot PR, then strips them from the tree

**Problem:**
A candidate doc dirty with user edits gets refresh edits applied on top;
selective staging stages the blend; checkout-back reverts the working tree
— the user's uncommitted work survives only inside the bot PR's commit.
The recorded dirtiness bit was never consumed by any rule.

**Fix:** Execution gate: candidates dirty at record time are read-only —
classified, reported as Recommended ("dirty at start — not touched"),
never edited, staged, or committed.

### 2
- status: applied
- severity: P2
- agent: adversarial-reviewer
- category: Git safety / deferred injection
- confidence: 70
- action_class: judgment
- file: plugins/cepa/commands/compound-refresh.md
- lines: 277-282
- title: Foreign-branch downgrade fired after Phase 3 — staged deletions primed for the owner's next commit

**Problem:**
Ownership was checked only in Phase 5's commit rules, so Phase 3 had
already mutated the foreign branch's tree and staged `git rm` deletions;
the owner's next plain `git commit` lands the refresh payload anyway —
injection delayed one commit, not prevented.

**Fix:** Ownership decided at the execution gate, before any write;
not-owned branch → report-only mode (no writes, no staging, Recommended
from investigation evidence alone). Invariant restated as "never mutates
work that isn't its own."
