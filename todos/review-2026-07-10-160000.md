### 6
- status: applied
- severity: P2
- agent: silent-failure-hunter + adversarial-reviewer
- category: Data loss ordering
- confidence: 80
- action_class: corroborated
- file: plugins/cepa/commands/compound-refresh.md
- lines: 35-41, 173-183
- title: Delete can run after a failed merge/successor write (Consolidate/Replace)

**Problem:**
Headless rule "record as Recommended and continue" is unscoped; applied
per-write it permits: merge into canonical fails → continue → delete subsumed
doc. Unique content gone from the tree, PR contains the deletion but not the
merge. Same trace for Replace ("...then delete the old doc") with a failed or
invalid successor. Inbound-link repointing ordered after delete.

**Fix:**
Atomicity rule: deletion is always the last step of its action and is gated
on verifying the surviving content exists on disk; any failed write makes the
ENTIRE action Recommended ("continue" = next doc, never the delete); repoint
inbound links before deleting; failed Replace validation → stale-mark old doc,
never delete.
