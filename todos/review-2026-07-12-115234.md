### 1
- status: deferred
- severity: P1
- agent: adversarial + feasibility
- category: Recall governance
- confidence: 100
- action_class: judgment
- file: docs/plans/2026-07-12-ob1-brain-integration.md
- title: Pending memories are dropped by recall by default — the loop no-ops until this is resolved

**Problem (code-verified):** `/writeback` stamps every cepa memory
`review_status=pending` + `requires_user_confirmation=true` (index.ts:509-512); `/recall`
drops exactly those unless `include_unconfirmed=true`, which defaults false
(index.ts:61,244). So compound writes, learnings-researcher recalls `[]`, and the whole
producer→consumer premise silently no-ops until a human confirms each memory.
**Decision needed:** (a) recall `include_unconfirmed=true` + treat all as evidence-only
under §7 (no human gate; unreviewed evidence flows in — §7 strips imperatives, not false
*evidence*); (b) cepa auto-PATCHes each write to `evidence_only` after writeback (auto,
uses the API's evidence state, still never instruction); (c) human reviews the pending
queue before recall (safest, throughput burden on a solo user).
