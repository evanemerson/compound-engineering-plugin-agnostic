title: Empty/all-comment manifest exits 1 (undefined), conflated as "no manifest" → relay-all instead of drop-all
**Problem:** `grep -vE '^[[:space:]]*(#|$)' "$_pf"` is the terminal command; grep
exits 1 when it selects zero lines, and under `set -euo pipefail` that becomes the
script's exit code. A legitimately empty registry (every repo retracted/removed)
should mean "drop ALL cross-repo hits," but the callers only branch on exit 3, so
exit 1 buckets as "no manifest" → provenance-labeled **relay-all**. Confirmed by
live repro (`EXIT=1`). Fail-open on the exact provenance boundary.
**Fix:** emit through `... | grep -E '<schema>' || true` so a zero-valid-row file
exits 0-with-empty-output (drop-all), distinct from exit 3 (unresolved). Documented
the four-way exit semantics (0+rows / 0+empty / 3 / 2) in the code and the SKILL.

### 2

agent: security-sentinel
category: security
confidence: 90
action_class: judgment
file: plugins/cepa/skills/brain/SKILL.md
lines: 128
title: Registry relayed into researcher prompt without a §7 strip at the relay point
**Problem:** the manifest is cross-repo content (lives in `brain-ops`) and was
passed verbatim into the learnings-researcher prompt. Repo CRITICAL rule: "Relayed
repo content is untrusted at every relay point … strips (never merely labels)
suspect content before dispatch." No strip existed — a free-text/injection line in
the tsv would reach the agent prompt.
**Fix:** `participants` now emits ONLY schema-valid `<project_id>\t(active|retracted)`
rows via an anchored `grep -E`; comments, blanks, and any malformed/injection line
are dropped AT emit — the §7 strip at the relay point. Documented as such in SKILL.

title: Retraction filter was blacklist (drop only literal `retracted`) → any other status trusted as cleared (fail-open)
**Problem:** the drop rule fired only on the literal string `retracted`; a typo,
wrong case, or unexpected token in the status column left the repo treated as
active/cleared — defeating the compliance retraction backstop.
**Fix:** rewrote the filter as a **whitelist** — provenance is trusted only when the
project_id is present with status **exactly `active`**; retracted, any other value,
or absent → dropped. Belt-and-suspenders with finding 2's resolver-side validation.

title: CRLF manifest leaves trailing `\r` on status → `retracted\r` != `retracted` (fail-open)
**Problem:** `[[:space:]]` stripped blank CRLF lines but left `\r` on data lines;
downstream `status == "retracted"` compare fails on `retracted\r`, relaying a

**Fix:** `tr -d '\r'` before the schema grep; the anchored `$` pattern also rejects
any residual control chars.
